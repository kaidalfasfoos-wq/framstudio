# Frame Studio — تطبيق إطارات صور + أكشنات لعدة صور

تطبيق أندرويد (Kotlin + Jetpack Compose) بثلاث ميزات رئيسية:

1. **معرض الإطارات**: استيراد صورة إطار PNG جاهزة (بخلفية شفافة) من الهاتف، أو تصميم إطار جديد داخل التطبيق (لون الحدود، السماكة، شكل الزوايا: بسيط/مدوّر/زخرفي، ونص أسفل الإطار).
2. **الأكشنات**: تسجيل مجموعة تعديلات (سطوع، تباين، تشبع، نسبة قص، إطار، توقيع نصي) وحفظها كأكشن قابل لإعادة الاستخدام — تماماً مثل أكشنات فوتوشوب.
3. **معالجة دفعة صور**: اختيار عدة صور من المعرض دفعة وحدة، اختيار أكشن، وتطبيقه على كل الصور تلقائياً مع شريط تقدّم، ثم حفظها في ألبوم "Frame Studio" بمعرض الصور ومشاركتها.

## أسهل طريقة تحصل فيها على APK جاهز بدون ما تنصّب شي (موصى فيها)

المشروع فيه ملف `.github/workflows/build.yml` يبني الـ APK تلقائياً على سيرفرات GitHub (مجاناً):

1. اعمل حساب على [github.com](https://github.com) إذا ما عندك، وسوّي مستودع (Repository) جديد **فاضي** — بدون README تلقائي.
2. ارفع محتويات مجلد `FrameStudio` كامل لهذا المستودع. أسهل طريقة: بالمتصفح اسحب وأفلت كل الملفات والمجلدات بصفحة "Upload files"، أو عبر Git:
   ```bash
   cd FrameStudio
   git init
   git add .
   git commit -m "Frame Studio initial commit"
   git branch -M main
   git remote add origin https://github.com/USERNAME/REPO.git
   git push -u origin main
   ```
3. روح لتبويب **Actions** بالمستودع — رح تلاقي عملية بناء (workflow) اسمها **Build APK** تشتغل تلقائياً بعد الرفع (تاخذ حوالي 3-5 دقائق).
4. لما تخلص وتصير علامة ✅ خضراء، افتحها واضغط على **Artifacts** بالأسفل، وحمّل ملف **FrameStudio-debug-apk** — جواه ملف `app-debug.apk` جاهز للتثبيت مباشرة على أي جهاز أندرويد (لازم تفعّل "تثبيت من مصادر غير معروفة" بإعدادات الجهاز).

هذا الـ APK نسخة **Debug** (للتجربة والاستخدام الشخصي مباشرة). إذا بدك تنشره على متجر أو توزّعه رسمياً، بتحتاج بناء نسخة **Release** موقّعة — أقدر أساعدك بهيك لما توصل لهاي المرحلة.

## طريقة الفتح والتشغيل (بديل: عبر Android Studio على جهازك)

1. افتح **Android Studio** (يفضّل أحدث إصدار — Koala أو أحدث).
2. اختر **Open** ووجّه إلى مجلد `FrameStudio` (المجلد الذي يحوي `settings.gradle.kts`).
3. انتظر حتى ينتهي **Gradle Sync** تلقائياً (يحتاج اتصال إنترنت لتحميل المكتبات أول مرة).
4. وصّل جهاز أندرويد فعلي أو شغّل محاكي (Emulator) بإصدار **Android 8.0 (API 26)** فأعلى.
5. اضغط **Run ▶** لتثبيت وتشغيل التطبيق.

## متطلبات تقنية

- **minSdk 26 / targetSdk 34**
- Kotlin 1.9.24، Jetpack Compose (BOM 2024.06.00)
- **Room** لتخزين الإطارات والأكشنات محلياً على الجهاز
- **Coil** لعرض الصور
- صلاحية قراءة الصور (`READ_MEDIA_IMAGES` على أندرويد 13+، وما يقابلها بالإصدارات الأقدم)

## هيكلة المشروع

```
app/src/main/java/com/framestudio/app/
├── data/          Room: FrameEntity, ActionEntity, DAOs, Database, Repository
├── imaging/        ImageProcessor (سطوع/تباين/تشبع/قص/تركيب إطار/حفظ)
│                    FrameRenderer (توليد بيتماب الإطار المصمَّم داخل التطبيق)
│                    BatchProcessor (تطبيق أكشن على قائمة صور مع تقدّم Flow)
├── viewmodel/      FrameViewModel / ActionViewModel / BatchViewModel
├── navigation/     NavGraph (كل شاشات التطبيق)
└── ui/screens/     Home, FrameGallery, FrameDesigner, ActionsList, ActionEditor,
                     BatchProcess, Result
```

## ملاحظات مهمة قبل الاستخدام الفعلي

- **أيقونة التطبيق**: وضعت أيقونة Adaptive بسيطة (مربع بلون العلامة). يُفضّل استبدالها بشعار احترافي عبر **Image Asset Studio** داخل Android Studio (Right click على `res` → New → Image Asset).
- **صلاحيات وقت التشغيل**: الكود جاهز من ناحية الـ Manifest، لكن يفضّل إضافة طلب صلاحية `READ_MEDIA_IMAGES` بشكل صريح عبر `ActivityResultContracts.RequestPermission()` قبل أول استخدام لمنتقي الصور، خاصة على أجهزة أندرويد 13 فما دون الـ Photo Picker الحديث (المستخدم حالياً `PickMultipleVisualMedia` لا يحتاج صلاحية على أندرويد 11+، لكن يفضّل اختباره على أجهزة حقيقية متعددة الإصدارات).
- **الأداء**: `ImageProcessor.decodeBitmap` يصغّر الصور تلقائياً (حد أقصى 2048 بكسل) لتفادي مشاكل الذاكرة عند معالجة عدد كبير من الصور دفعة وحدة.
- **التوسّع المستقبلي**: يمكن بسهولة إضافة فلاتر جاهزة (Instagram-style)، أو رفع الإطارات لمشاركتها بين المستخدمين، أو ربط الحفظ بـ Google Drive.
